# Repo structure

Codifies plan §3.10 (Documentation system) for this meta-repo and as a reference for any project applying an L3 template.

## Tree

```
cascade-system/
├── README.md                       # index — links everything below
├── retros/
│   └── sprint-N.md                 # sprint retrospectives
├── queue/
│   └── pending-review.md           # drained by /sprint-review
├── refs/                           # vendored reference repos (gitignored)
└── docs/
    ├── structure.md                # this file — directory conventions
    ├── makefile-manual.md          # all `make` commands documented (when Makefile exists)
    ├── technical.md                # implementation explanations (cross-cutting)
    ├── architecture/               # reserved for system-overview + cross-Cascade flow diagrams
    │   ├── *.excalidraw            # conceptual, hand-drawn feel
    │   └── *.mmd                   # structured Mermaid diagrams
    ├── decisions/
    │   ├── INDEX.md                # ADR index
    │   └── ADR-NNN-<slug>.md       # one per significant decision
    ├── handoffs/
    │   ├── INDEX.md                # handoff prompt index
    │   └── <from-cascade-id>-<to-sprint-or-vertical>.md
    └── reviews/                    # peer-Cascade review reports (added per ADR-007)
        └── <sprint-or-vertical-slug>-review.md
```

**Meta-repo note**: the cascade-system meta-repo does **not** run its own `phases.yaml` — it is a horizontal meta-project, not a phase-driven vertical. Therefore `docs/prompts/stages/` does **not** appear in this tree. Downstream projects bootstrapped via `/start-project` do get `docs/prompts/stages/` (see `## When applying L3 templates to a downstream project`). Per ADR-010.

## Placement rules

1. **Every artifact lives in its canonical subdirectory.** `docs/` root holds only `structure.md`, `makefile-manual.md`, `technical.md`. New top-level docs require an ADR.
2. **New subdirectory category** → ADR first (precedent: ADR-002 for `docs/handoffs/`).
3. **ADRs are immutable** once `Status: Accepted`. Override via `Status: Superseded by ADR-MMM`, never delete. Handoffs and retros follow the same retention discipline per ADR-011.
4. **Indexes** (`INDEX.md` in `decisions/` and `handoffs/`) are regenerated by `/docs-refresh` (M1.11). Update manually until then.
5. **Retros live at repo root `retros/`**, not under `docs/retros/`. Per plan §3.10 the project's retros are at `<project>/docs/retros/`, but **for this meta-repo retros are cross-project archives** and live one level up — they aren't strictly project documentation, they aggregate learnings from other projects.
6. **Empty subdirs carry rationale** (ADR-010): any subdir that is intentionally present-but-empty (e.g., `architecture/` until a diagram lands) gets either a `.gitkeep` + README stub, or is removed from this tree. Orphaned empty dirs are flagged by `/docs-refresh` as warnings.
7. **ADR numbering** is claimed by editing `INDEX.md` first, per ADR-009 + `document-as-you-go` `## Numbering allocation`.

## When applying L3 templates to a downstream project

Downstream projects (e.g., thesis) get the same `docs/` structure:

```
<project>/
├── .windsurf/
│   ├── rules/                      # L2 overrides only
│   └── phases.yaml                 # copied from L3 template
└── docs/
    ├── structure.md                # copy of this file
    ├── architecture/
    ├── decisions/
    ├── retros/                     # project-local retros (NOT in repo root)
    └── prompts/stages/
```

The difference: in downstream projects, `docs/retros/` is the canonical location. In this meta-repo, `retros/` is at the root because the retros aggregate cross-project learnings.

## Validators

`/docs-refresh` (M1.11) will:

- Reject artifacts in `docs/` root that aren't `structure.md`, `makefile-manual.md`, or `technical.md`
- Reject ADRs not matching `ADR-NNN-<kebab-slug>.md`
- Reject handoff files not matching `<cascade-id>-<sprint-or-vertical>.md`
- Regenerate `INDEX.md` files

Until M1.11 lands, placement discipline is manual.
